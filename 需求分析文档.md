# 0. 开发目标

实现一个 **前后端分离的开发者支付管理平台**。

系统必须支持：

- 开发者注册
- 支付订单创建
- 支付二维码生成
- 自动手续费结算
- 钱包系统
- 提现系统
- 支付回调

------

------

# 4. 支付系统（核心）

------

## Task 4.1 创建支付订单

```
POST /api/pay/create
```

流程：

```
验证 pay_key
验证签名
检查订单是否存在
生成 platform_order_no
创建订单
返回支付二维码URL
```

------

------

## Task 4.2 签名验证模块

创建：

```
utils/signature.py
```

实现：

```
generate_sign()
verify_sign()
```

------

------

## Task 4.3 二维码生成服务

创建：

```
services/payment_service.py
```

返回：

```
mock pay_url
```

（初期无需真实支付）

# 5. 支付成功处理

------

## Task 5.1 支付回调入口（模拟）

```
POST /api/pay/notify
```

逻辑：

```
订单状态 → SUCCESS
计算手续费
更新钱包余额
记录支付时间
```

------

------

## Task 5.2 幂等处理（必须）

要求：

```
SUCCESS订单不可重复处理
```

使用：

```
数据库事务 + 状态判断
```
